# Support & Bug Reports

How to get help and report issues with SourceLens.

## Getting Help

### Documentation
- Check the [README](./README.md) for usage instructions
- See [TROUBLESHOOTING.md](./TROUBLESHOOTING.md) for common issues
- Review [PRIVACY_POLICY.md](./PRIVACY_POLICY.md) for privacy questions

### Reporting Bugs

If you encounter a bug or issue:

1. **Check existing issues:**
   - Search the repository's issue tracker
   - Check if your issue has already been reported

2. **Gather information:**
   - **Browser version:** `chrome://version/`
   - **Extension version:** Settings → About section (or `chrome://extensions/`)
   - **Chrome version:** Help → About Google Chrome
   - **Steps to reproduce:** What actions led to the issue?
   - **Expected behavior:** What should have happened?
   - **Actual behavior:** What actually happened?
   - **Console errors:** Press `F12` → Console tab → Copy any errors
   - **Screenshots:** If applicable, include screenshots

3. **Create an issue:**
   - Use the bug report template
   - Include all gathered information
   - Be clear and concise

### What Information to Include

**Required:**
- Browser version (Chrome/Chromium version)
- Extension version
- Steps to reproduce the issue
- Expected vs actual behavior

**Helpful:**
- Console error messages
- Screenshots or screen recordings
- Whether the issue is reproducible
- Any error messages shown in the UI

**Do NOT include:**
- ❌ Personal information
- ❌ ChatGPT conversation content
- ❌ Account credentials or tokens
- ❌ Sensitive data

### Feature Requests

We welcome feature requests! When suggesting a feature:

1. **Describe the feature:**
   - What problem does it solve?
   - How would it work?
   - Why is it useful?

2. **Consider privacy:**
   - Ensure the feature aligns with our privacy-first approach
   - No data collection or external requests

3. **Create an issue:**
   - Use the feature request template
   - Include your reasoning

## Security Issues

If you discover a security vulnerability:

1. **Do NOT open a public issue**
2. **Email the maintainer directly** (see repository for contact)
3. **Include:**
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if you have one)

We take security seriously and will respond promptly.

## Response Time

We aim to respond to issues within:
- **Critical bugs:** 1-2 business days
- **Feature requests:** 1 week
- **General questions:** 3-5 business days

Please be patient—this is a volunteer/maintained project.

## Before Reporting

1. ✅ Check if the issue is already reported
2. ✅ Read the troubleshooting guide
3. ✅ Try the suggested solutions
4. ✅ Update to the latest version
5. ✅ Clear cache and data if applicable

## Issue Templates

When creating an issue, please use the appropriate template:

- **Bug Report:** For bugs and unexpected behavior
- **Feature Request:** For new features or improvements
- **Question:** For general questions or clarification
